# 竞品分析

> 版本: 1.0 · 2026.04

## 一、竞品全景

OfferBot 所处的竞争格局可以分为四个象限：

```
                    AI 智能化程度高
                         ↑
                         │
      OfferBot ──────────┤──────── AI 简历工具
      (全流程 Agent)      │         (超级简历/智联AI)
                         │
                         │
      自动投递脚本 ──────┤──────── 招聘平台
      (Boss自动投/猎聘助手)│         (Boss/猎聘/智联)
                         │
                         ↓
                    AI 智能化程度低
      ←──────────────────┼──────────────────→
      求职者工具                        招聘平台
```

## 二、直接竞品对比

### 2.1 招聘平台内置 AI 功能

| 维度 | Boss 直聘 | 猎聘 | 智联招聘 | OfferBot |
|------|-----------|------|----------|----------|
| 岗位推荐 | 协同过滤 + 简单匹配 | 猎头推荐 + 算法 | 关键词匹配 | 知识图谱 + 7 维度分析 |
| 匹配分析 | 无 | 简单匹配度百分比 | 无 | 技能/经验/薪资/学历/职责/位置/语义 7 维度 |
| 打招呼语 | 平台模板 | 平台模板 | 无 | 基于用户画像 + JD 个性化生成 |
| 面试追踪 | 基础状态（已读/已回复） | 基础状态 | 基础状态 | 10 阶段状态机 + 漏斗分析 |
| 用户画像 | 简历字段 | 简历字段 | 简历字段 | 10 维度记忆画像（含性格/风格/规划） |
| 数据归属 | 平台所有 | 平台所有 | 平台所有 | 用户本地所有 |
| 跨平台 | 仅 Boss | 仅猎聘 | 仅智联 | 猎聘 + 智联（可扩展） |

**核心差异：** 招聘平台的 AI 服务于平台的商业目标（提高平台活跃度和付费转化），OfferBot 的 AI 服务于用户的求职目标（提高匹配效率和面试成功率）。

### 2.2 AI 简历/求职工具

| 维度 | 超级简历 | 牛客 AI | 面试鸭 | OfferBot |
|------|----------|---------|--------|----------|
| 核心功能 | 简历模板 + AI 润色 | 面试题库 + AI 模拟 | AI 模拟面试 | 全流程 Agent |
| 覆盖环节 | 简历 | 面试准备 | 面试准备 | 岗位发现→匹配→投递→面试→追踪 |
| 个性化程度 | 低（通用模板） | 中（按岗位分类） | 中（按岗位分类） | 高（基于用户记忆画像） |
| 数据积累 | 无 | 刷题记录 | 面试记录 | 完整求职生命周期数据 |
| 部署方式 | SaaS | SaaS | SaaS | 本地部署（开源） |

**核心差异：** 这些工具解决的是求职链路中的单个环节，OfferBot 解决的是全链路的智能编排。

### 2.3 自动投递脚本/插件

| 维度 | Boss 自动投 | 猎聘助手 | OfferBot |
|------|-------------|----------|----------|
| 投递方式 | 无差别批量投递 | 简单条件过滤后投递 | AI 匹配分析后精准投递 |
| 匹配能力 | 无 | 关键词过滤 | LLM 多维度分析 |
| 打招呼语 | 固定模板 | 固定模板 | 个性化生成 |
| 风控 | 无/简单延迟 | 简单延迟 | 反检测模块 + 人类行为模拟 |
| 数据分析 | 无 | 简单统计 | 面试漏斗 + 转化率分析 |
| 合规风险 | 高（易被封号） | 中 | 低（仅爬取模式 + 用户确认投递） |

**核心差异：** 自动投递脚本追求"量"，OfferBot 追求"质"。无差别投递不仅效果差，还有被平台封禁的风险。

## 三、OfferBot 的差异化优势

### 3.1 知识图谱驱动的岗位理解

传统方案用关键词匹配或向量相似度做岗位推荐，OfferBot 用 LightRAG 构建岗位知识图谱：

```
传统方案:  用户简历 → Embedding → 余弦相似度 → 排序
           问题: 语义粒度粗，"Python 开发"和"AI 工程师"可能相似度很高但实际不匹配

OfferBot:  岗位 JD → 实体提取(技能/公司/城市/薪资/职责) → 知识图谱
           用户画像 → 多维度匹配(技能交集/经验区间/薪资重叠/...) → 量化评分
           优势: 可以回答"哪些公司在招 Agent 方向"、"AI 岗位最需要什么技能"等聚合问题
```

### 3.2 记忆画像系统

招聘平台只知道用户的简历字段（学历、工作经历、技能标签），OfferBot 通过持续对话积累 10 个维度的用户认知：

| 维度 | 招聘平台知道的 | OfferBot 额外知道的 |
|------|---------------|-------------------|
| 技能 | 技能标签列表 | 每个技能的熟练度、项目中的实际应用场景 |
| 经验 | 工作年限 | 具体做过什么、踩过什么坑、有什么心得 |
| 目标 | 期望薪资/城市 | 为什么想换工作、看重什么、不能接受什么 |
| 性格 | 无 | 沟通风格、决策偏好、价值观 |
| 规划 | 无 | 3-5 年职业规划、学习方向、成长目标 |

这些信息让 OfferBot 能生成真正个性化的内容（打招呼语、面试建议），而不是千篇一律的模板。

### 3.3 数据主权

| 方案 | 数据存储 | 用户控制 |
|------|----------|----------|
| 招聘平台 | 平台服务器 | 用户无法导出完整数据 |
| SaaS 工具 | 云端 | 依赖服务商的隐私政策 |
| OfferBot | 用户本地 SQLite + 文件 | 用户完全控制，可随时删除 |

对于简历、求职意向、面试记录这类敏感数据，本地存储是最强的信任基础。

## 四、竞争壁垒分析

| 壁垒类型 | OfferBot 的壁垒 | 可复制性 |
|----------|-----------------|----------|
| 技术壁垒 | 知识图谱 + 记忆系统 + ReAct Agent 架构 | 中（需要 AI 工程能力） |
| 数据壁垒 | 用户本地数据积累（记忆画像越用越准） | 高（数据在用户本地，迁移成本低） |
| 生态壁垒 | 开源社区 + Skills 插件体系 | 中（需要社区运营） |
| 体验壁垒 | 对话式交互 + 场景驱动设计 | 低（交互模式可模仿） |

**诚实的判断：** OfferBot 的核心壁垒不在于某个单点技术，而在于"全流程 + 理解用户 + 本地部署"这个组合。单独拆开每个能力都可以被复制，但组合在一起形成的用户体验是有门槛的。

## 五、竞争策略

1. **不与平台竞争，做平台的增强层** — OfferBot 不拥有岗位数据，而是帮用户更好地使用现有平台
2. **开源建立信任** — 用户可以审查代码，确认数据安全；社区贡献扩展平台支持
3. **记忆系统建立粘性** — 用得越久，AI 越了解用户，迁移成本自然提高
4. **场景深度而非广度** — 先把求职这一个场景做透，不急于扩展到其他领域
