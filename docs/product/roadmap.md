### 关键决策记录

| 决策 | 选择 | 理由 |

| 前端 | Chainlit | 开箱即用的对话 UI，快速验证产品 |

## 三、V1.5 — 体验优化（2026 Q2，规划中）

### 目标
优化核心体验，扩展平台支持，提升系统性能。

### 功能清单

| 模块 | 功能 | 优先级 | 说明 |
|------|------|--------|------|
| 平台 | Boss 直聘支持 | P1 | 需要解决封禁问题，可能需要新的反检测策略 |
| 路由 | 语义路由层 | P1 | 工具按需注入，减少 token 消耗 |
| UI | 数据可视化面板 | P1 | 面试漏斗图表、岗位分布图、趋势图 |
| UI | 批量操作优化 | P1 | 批量匹配分析、批量投递的进度展示 |
| 性能 | LLM 调用优化 | P1 | 程序化维度不消耗 token，减少 LLM 调用次数 |
| 性能 | 图谱查询缓存 | P2 | 相同查询复用结果，减少 LightRAG 调用 |
| 体验 | 对话历史管理 | P2 | 多会话切换、历史搜索 |
| 体验 | 错误恢复优化 | P2 | 任务中断后的优雅恢复 |

### 优先级排序逻辑
- Boss 直聘是最大的招聘平台，不支持会损失大量用户
- 语义路由直接影响 token 成本和响应速度
- 数据可视化是用户反馈最多的需求

## 六、优先级排序原则

1. **用户价值优先** — 直接解决用户痛点的功能优先于技术优化
2. **闭环优先** — 先跑通完整流程，再优化单个环节
3. **数据驱动** — 有数据支撑的需求优先于直觉判断
4. **成本可控** — 优先选择 token 消耗低、实现成本低的方案
5. **可逆性** — 优先做可以快速回滚的改动，不可逆的决策慎重

## 七、风险与应对

| 风险 | 影响 | 概率 | 应对策略 |
|------|------|------|----------|
| 招聘平台封禁 | 数据采集能力受限 | 高 | 多平台分散风险；scrapeOnly 模式降低频率 |
| LLM API 成本 | 用户使用成本高 | 中 | 程序化维度替代 LLM；缓存优化；支持本地模型 |
| 开源竞品出现 | 用户分流 | 中 | 记忆系统建立粘性；社区运营建立生态 |
| 数据隐私法规 | 合规风险 | 低 | 本地部署，数据不出设备；开源可审计 |
