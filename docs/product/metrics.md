# 数据指标体系

> 版本: 1.0 · 2026.04

## 一、北极星指标

**用户求职效率提升倍数** = 传统方式完成同等求职动作的时间 / 使用 OfferBot 的时间

目标：3x 以上（即原来 3 小时的工作量，用 OfferBot 1 小时完成）。

衡量方式：
- 传统方式：手动浏览 50 个岗位 + 分析匹配 + 写打招呼语 + 投递 ≈ 3 小时
- OfferBot：一键爬取 + AI 筛选 Top 10 + 自动生成打招呼语 + 确认投递 ≈ 30-60 分钟

---

## 二、核心指标框架

采用 AARRR 海盗指标模型，适配求职场景：

```
Acquisition（获取）→ Activation（激活）→ Retention（留存）→ Revenue（变现）→ Referral（推荐）
      │                    │                  │                  │                │
  开源下载量           首次完成搜索        日活/周活          N/A（开源）      GitHub Star
  GitHub 访问          首次匹配分析        求职周期内          社区贡献        社区分享
                       首次投递            持续使用天数
```

## 三、分模块指标

### 3.1 Agent 核心

| 指标 | 定义 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 意图识别准确率 | Agent 正确理解用户意图的比例 | > 90% | 抽样人工评估 |
| 工具路由准确率 | Agent 选择正确工具的比例 | > 85% | 执行轨迹分析 |
| 单轮响应时间 | 从用户发送到 Agent 回复的时间 | < 15s（纯对话）< 30s（含工具调用） | 系统日志 |
| 工具调用成功率 | 工具执行成功的比例 | > 95% | 执行轨迹统计 |
| 平均对话轮数 | 完成一个任务的平均对话轮数 | < 5 轮 | 对话日志分析 |

### 3.2 岗位数据采集

| 指标 | 定义 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 爬取成功率 | 成功爬取的岗位数 / 总发现岗位数 | > 90% | getjob 统计 |
| JD 获取率 | 成功获取完整 JD 的岗位比例 | > 80% | 本地数据库统计 |
| 数据同步延迟 | 爬取完成到本地可查询的时间 | < 30s | 系统日志 |
| 图谱构建成功率 | 成功插入知识图谱的岗位比例 | > 95% | LightRAG 日志 |

### 3.3 匹配分析

| 指标 | 定义 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 匹配准确率 | 高匹配度岗位中用户认可的比例 | > 70% | 用户反馈（投递/跳过） |
| 分析耗时 | 单岗位匹配分析时间 | < 15s | 系统日志 |
| 批量分析吞吐 | 每分钟可分析的岗位数 | > 3 个 | 系统日志 |
| 技能识别准确率 | 正确识别的技能数 / JD 中实际技能数 | > 85% | 抽样人工评估 |

### 3.4 投递与沟通

| 指标 | 定义 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 打招呼语差异化率 | 不同岗位生成的打招呼语内容差异度 | > 80% | 文本相似度计算 |
| 投递成功率 | 成功发送打招呼语的比例 | > 95% | 投递记录统计 |
| 已读率 | 投递后被 HR 查看的比例 | 基线对比 | 面试追踪数据 |
| 回复率 | 投递后收到 HR 回复的比例 | 基线对比 | 面试追踪数据 |

### 3.5 面试漏斗

| 指标 | 定义 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 投递→已读转化率 | 已读数 / 投递数 | > 50% | interview_tracking |
| 已读→回复转化率 | 回复数 / 已读数 | > 20% | interview_tracking |
| 回复→约面转化率 | 约面数 / 回复数 | > 50% | interview_tracking |
| 约面→offer 转化率 | offer 数 / 约面数 | > 15% | interview_tracking |
| 平均求职周期 | 从首次投递到拿到 offer 的天数 | 参考值 | interview_tracking |

### 3.6 记忆系统

| 指标 | 定义 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 记忆提取准确率 | 正确提取的信息条数 / 对话中可提取的信息总数 | > 85% | 抽样人工评估 |
| 记忆覆盖度 | 有内容的记忆维度数 / 总维度数（10） | > 60%（5 次对话后） | 记忆文件统计 |
| 记忆应用率 | 生成任务中实际使用记忆数据的比例 | > 80% | 执行轨迹分析 |
| 记忆冲突率 | 新提取信息与已有记忆冲突的比例 | < 10% | 记忆系统日志 |

### 3.7 知识图谱（LightRAG）

| 指标 | 定义 | 目标值 | 衡量方式 |
|------|------|--------|----------|
| 实体提取准确率 | 正确提取的实体数 / JD 中实际实体数 | > 80% | 抽样评估 |
| 检索相关性 | 返回结果中与查询相关的比例 | > 70% | 抽样评估 |
| 检索响应时间 | 从发起查询到返回结果的时间 | < 5s | 系统日志 |
| 图谱覆盖率 | 已图谱化的岗位数 / 有 JD 的岗位总数 | > 95% | 数据库统计 |

## 五、指标驱动的产品迭代

| 指标异常 | 可能原因 | 产品动作 |
|----------|----------|----------|
| 意图识别准确率 < 80% | System Prompt 不够清晰 | 优化 Prompt + 增加场景示例 |
| 匹配准确率 < 60% | 匹配维度权重不合理 | 调整权重 + 增加用户反馈机制 |
| 回复率持续低于 15% | 打招呼语质量不够 | 优化生成 Prompt + 分析高回复率样本 |
| 记忆提取准确率 < 70% | 提取 Prompt 不够精确 | 优化 MemoryExtractor Prompt |
| 检索相关性 < 60% | 图谱实体提取不准 | 优化 LightRAG 实体类型和提取规则 |
