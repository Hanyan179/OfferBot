## 场景描述

用户看到一个具体岗位（或提供了 JD），需要一份针对性简历。不是重写简历，是基于基准简历做精准微调。

## 核心原则：基准不可变 + 表达可调

### 绝对不可变（事实层）
- 姓名、联系方式、学历、学校
- 工作经历的公司名、职位、时间段
- 项目名称、实际使用的技术栈
- 工作年限

### 可调整（表达层）
- **个人简介/自我评价**：根据岗位方向调整侧重点
- **技能排序**：把与 JD 匹配的技能排在前面
- **工作经历描述**：同一段经历，强调与目标岗位相关的部分
- **项目亮点**：选择性突出与 JD 匹配的成果
- **核心亮点**：根据岗位需求重新组织

### 不可以做的
- 不能编造不存在的经历或项目
- 不能虚报技术栈（没用过的不能写）
- 不能夸大数据指标

## 执行逻辑

1. **获取基准简历**：调用 `get_user_profile()` 获取完整的基准简历数据
2. **获取用户画像**：调用 `get_user_cognitive_model()` 了解用户的核心优势和特点
3. **获取关键记忆**：
   - `get_memory(category="key_points")` — 项目细节、技术深度
   - `get_memory(category="career_planning")` — 职业方向、核心诉求
4. **分析目标岗位**：
   - 如果用户提供了 JD，提取关键要求（技能、经验、职责）
   - 如果只说了方向（如"AI Agent 方向"），基于记忆中的岗位认知理解需求
5. **生成定制版本**：
   - 保持事实层不变
   - 调整表达层，让简历"说对方想听的话"
   - 输出完整的定制简历内容（Markdown 格式）
6. **标注调整点**：告诉用户"我调整了哪些部分、为什么这样调"

## 输出格式

直接输出 Markdown 格式的完整简历，不需要保存到数据库。用户满意后可以自行下载或复制。

## 注意事项

- 定制简历是一次性输出，不覆盖基准简历
- 如果基准简历信息不完整，先提醒用户补充
- 每次定制都应该说明"基于你的基准简历，针对 XX 岗位做了以下调整"
